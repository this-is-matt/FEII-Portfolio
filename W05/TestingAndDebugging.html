Debugger keyword is used to send the console information. It also provides a breakpoint.

Throwing errors can be done with native error objects, but we can also use the throw statement to create our own errors. Throwing customized errors is more ideal than non-ideal responses like NaN as an output.

Thrown errors terminate the program, which doesn't look too good.

Try, Catch, Finally blocks are used to throw erros and handle them gracefully.

Test Driven Development: write test first, then code: 1.Write tests (that initially fail)
2.Write code to pass the tests
3.Refactor the code
4.Test refactored code
5.Write more tests for new features
