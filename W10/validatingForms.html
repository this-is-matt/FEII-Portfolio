• two types of client-sode validation: built in validation, and Javascript.
  • built in includes: required, minlength and maxlength. min and max, type, and regex patterns.
  • When an element is valid with built in it matches the :valid css pseudoclass, if the user submits, the breowser won't stop it.
  • The text area doesn't support a pattern attribute.
  • best practice for character length requriements is to use the attributes, but to also provide a character count using JS

    • Validating using JS

  • It's called the Constraint Validation API
  • This API affects the following elements: Button, fieldset, inout, output, select, textarea
  • The API makes the validationMessage, willvalidate, and Validity properties available for the above listed elements.
  • Validity returns an object with many properties, some of the common ones include tooLong, tooShort, typeMismatch, :invalid, etc.
  • There are various methods as well.
