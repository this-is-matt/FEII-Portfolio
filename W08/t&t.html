Transform and Translate

  • Translate takes two values, x and y as px
  • Transforms don't work on inline tags.
  • Scale takes two values as well. If you only provide one it will use the same value for both x and y
  • transforms adjust from the center of the element, This can be changed with transform-origin.
  • if doing two or more transforms, they have to be in one property, not two.
  • rotate is another option. Declared with deg, radians,grad, or turns - use degrees.
  • Transforms are executed in the same order they appear in the property.
  • lastley is skew, which accepts two values x and y. if the second parameter is ommitted it will only apply the x axis.
  • make sure you order your transforms in the right order, if you rotate first, the axis for the other ones changes as well.
  • Transition properties should be declared on the defualt properties.
  • Apparently 200 miliseconds is the ideal transition-duration.
  • transition-timing-function allows you to adjust how the transition happens. To define your own, use cubic-bezier.
  • There is a transition shorthand: property, duration, timing-function, and delay
  • Animations allow you to create more intricate animatios by describing each key frame. DO this using the keyframes property.
  • The animation property takes as its value a space-separated list of values for the longhand animation-name, animation-duration, animation-timing-function, animation-delay, animation-iteration-count, animation-direction, animation-fill-mode, and animation-play-state properties
