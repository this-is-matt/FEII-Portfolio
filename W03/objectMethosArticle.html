Object Methods Article

Article

Objects often are created to represent real world objects like users, orders, etc.

let user = {
name: "John",
age: 30
};

We can use functions to tell objects how to act. Functions tied to objects are called methods and are declared like this:
"objectName.functionName = function(){..."
we can also call a method like this: "objectName.FunctionName();"

One can also declare methods through an object literal:

user = {
sayHi() { // same as "sayHi: function(){...}"
alert("Hello");
}
};

Generally THe object literal notation is preferred.

this.

"this." is used to access the object and it's variables, methods, etc.: "this.name"

using "this" is more reliable than the alternative, which is to reference the out variable (object name)

"this." can be used in unbound function(not-methods) and when called by a method definition or something else withing the object it will work properly.

if a function is called that utilizes "this" outside of an object it will reference the global object(window in a browser). in strict mode it will throw an error.

"this" acts differently in arrow functions. Do more research if you intend to use it. It references the outer object.
